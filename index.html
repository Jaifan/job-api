Job API with JsonWebToken based Authentication

For Experience the API , You need to use POSTMAN reqest response system.


System design with two routes :
1) Authentication with (Register , Login) feature
2) Job list access control with JWT Authentication. For specific user have custom job list.


Resgister/Login Schema : {"name" , "email" , "password"} and Password is encrypted by bcrypt.

Register gateway(POST) GET also possible for Development NOW "api/v1/auth/register"

Login gateway(POST) "api/v1/auth/login"

After Login , User get a JWT bearer token for Authentication for each api request.

Job Schema : {"company", "position" , "status" , "createdBy(userID)"}

Job list gateway (GET , POST) "api/v1/jobs/" (JWT token require for Access!!)

Job Update (PATCH , GET) "api/v1/jobs/:id" for Specific Job any update.
